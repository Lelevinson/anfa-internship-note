# General IT Support Triage

tags: #internship #troubleshooting #support #triage

## Purpose

Use this before jumping into a fix. It helps you collect the right information and avoid guessing.

## Basic triage flow

```text
1. What exactly failed?
2. Who is affected?
3. Where is it happening?
4. When did it start?
5. Is it one device, one room, or everyone?
6. What changed recently?
7. What safe checks can I do?
8. What evidence should I collect before escalating?
```

## 1. Confirm the exact issue

Ask or check:

- Cannot open?
- Login failed?
- Button failed?
- Report/export failed?
- Cannot connect?
- Cannot print?
- Cannot share screen?
- File cannot copy?

## 2. Confirm scope

This decides whether the issue is probably device-specific, room-specific, account-specific, or system-wide.

Ask:

- Only this user?
- Only this PC?
- Only this room?
- Only this network?
- Everyone?

## 3. Confirm environment

Record:

- PC/laptop/tablet model if relevant
- room number
- system name
- browser used
- exact URL if safe to record
- wired network or Wi-Fi
- account/user type, without storing password

## 4. Safe checks

Depending on the problem:

- Browser/system issue: see [[IE Mode and Legacy Systems]]
- Printer issue: see [[Printer Spooler]]
- Wi-Fi/AP issue: see [[WiFi and AP Basic Checks]]
- Meeting room display issue: see [[Meeting Room Display and HDMI]]
- Windows service/log issue: see [[Windows Services and Event Viewer]]

## 5. Evidence to collect before escalating

- screenshot or photo
- exact error message
- user/device/room affected
- time problem happened
- steps to reproduce
- what was already tried
- whether it happens on another PC

## Important rule

Do not guess passwords, permissions, or system meanings. If the fix requires admin access, internal permission, or production-system change, collect information and ask mentor/senior.
