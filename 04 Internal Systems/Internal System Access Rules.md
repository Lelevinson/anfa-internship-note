# Internal System Access Rules

tags: #internship #internal-system #access #browser #permissions

## Purpose

This note keeps the important access rules in one place so they are not scattered across daily logs.

## Browser / IE Mode rules

Current understanding:

- YINXUN / 音訊 uses IE Mode.
- Other old internal web pages should not automatically use IE Mode.
- Some older pages may need to be opened through the correct internal host/address instead.
- Compatibility mode may also need to be enabled for some legacy behavior.

See [[IE Mode and Legacy Systems]].

## Network / permission rules

Current understanding:

- Some database/file access requires being on the correct company network first.
- Some access requires permission / 權限 approval.
- If access is denied, do not assume it is a password problem.
- Confirm whether the issue is network, account, permission, browser, or system-specific.

See [[Windows Domain AD and Permissions]].

## What not to store in this vault

Do not store:

- passwords
- real customer data
- screenshots with personal information
- private medical details
- full sensitive internal documents
- anything your mentor/company would not want outside the company system

## What is safe to store

Usually safe:

- general process notes
- generic troubleshooting steps
- system names without private records
- learning topics
- non-sensitive equipment locations
- questions to ask mentor/senior
